# Contributing to SOSU

Thank you for your interest in contributing to SOSU! We welcome bug reports, feature requests, and PRs. Please follow these guidelines to make collaboration smooth.

## How to Contribute
- Fork the repository and create a feature branch: `git checkout -b feature/your-feature`.
- Keep changes small and focused (prefer one change per PR).
- Write a clear PR description with motivation and testing steps.

## Code Style
- Use React functional components and hooks.
- Keep components small and focused; extract utility functions where appropriate.
- Run linting and formatters (if present) before opening a PR.

## Tests & Manual QA
- Provide manual testing steps in the PR description.
- If you add non-trivial logic, include unit tests where appropriate.

## Reviewing & Merging
- PRs will be reviewed by maintainers; please respond to review feedback promptly.
- Maintain backwards-compatible changes unless marked as breaking.

Thanks again — we appreciate your time and help! ❤️
